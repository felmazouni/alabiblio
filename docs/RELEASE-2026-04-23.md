# Release 2026-04-23

## Alcance

- Despliegue del estado endurecido mas reciente validado en preview y promovido a produccion sin alterar ni ocultar ratings.
- Cierre de deuda operativa para que el estado publicado quede trazable y reproducible desde repositorio.

## Endurecimiento incluido

- Ratings visibles y honestos: se mantienen en producto como valoraciones internas de AlaBiblio, con etiquetas de procedencia y estado de muestra.
- Filtros de horario retirados de la UI publica y neutralizados en query parsing para no prometer semantica de dias/franjas que no estaba cerrada con suficiente honestidad.
- Distancia heuristica etiquetada como aproximada y distancia por proximidad deshabilitada cuando no existe ubicacion real del usuario.
- Copy publica corregida para eliminar claims engañosos de realtime.
- Filtro publico de transporte alineado con modos realmente visibles; el backend filtra con `hasVisibleTransportMode(...)`.
- `/api/health` endurecido como health real con checks de base de datos, centros y snapshots de transporte.

## Riesgos mitigados

- Se evita vender filtrado horario fino no defendible.
- Se evita presentar ratings como reseñas externas o como muestra madura cuando todavia no lo es.
- Se evita mostrar distancia utilizable sin ubicacion real.
- Se evita devolver centros por modo de transporte si ese modo no tiene opcion visible/publicable en el payload.
- Se evita un falso positivo de regresion por cache de assets tras deploy de Cloudflare; la validacion final se hizo con recarga fresca y API real.

## Riesgos abiertos

- La semantica completa de horarios sigue necesitando auditoria funcional dura antes de reabrir filtros por dias o franjas.
- Persisten centros con `needsManualReview` en horario o con transporte parcial publicado desde texto oficial cuando no existe alternativa estructurada mejor.
- El estado de coche sigue siendo heuristico y se mantiene solo como SER + accion de abrir ruta, sin motor de routing.

- Ratings visibles y etiquetados como internos.
- Sin pestana ni filtros publicos de horarios en la UI.
- Sin distancia engañosa cuando falta ubicacion.
- Filtro de transporte coherente con transporte visible/publicado.
- Health de produccion devolviendo estado real y checks estructurados.
